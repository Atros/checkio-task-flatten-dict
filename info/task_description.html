Nicola likes to categorize everything. He categorized all reagents in his laboratory, books in the library and notes on the desk. But then he had known about python dictionaries and categorized all configurations for Sophia drones. Now these files are looked as deep nested structure, but Sophia don't like this. Let's help Sophia to flatten these dictionaries.

Python dictionaries are a convenient data type to store and process configurations. It allows to store data by keys and to create nested structures. You are given a dictionary where keys are strings and values are strings or dictionaries. The goal is flatten the dictionary, but save all structures in the keys. The result should be the new dictionary without nested dictionaries. The keys should contains a path that contains parent keys from the original dictionary. The keys in the path are separated by "/". If a value is an empty dictionary, then it should be replaced by an empty string (""). Let's look at example. 

{
    "name": {
        "first": "One",
        "last": "Drone"
    },
    "job": "scout",
    "recent": {},
    "additional": {
        "place": {
            "zone": "1",
            "cell": "2"}
    }
}

The result will be: 

{"name/first": "One",           #one parent
 "name/last": "Drone",
 "job": "scout",                #root key
 "recent": "",                  #empty dict
 "additional/place/zone": "1",  #third level
 "additional/place/cell": "2"}

Sophia already have written the code for this task, but it has a bug. Your should find and fix this bug.

Input: An original dictionary as a dict.

Output: The flatten dictionary as a dict.

Example: 

flatten({"key": "value"}) == {"key": "value"}
flatten({"key": {"deeper": {"more": {"enough": "value"}}}}) == {"key/deeper/more/enough": "value"}
flatten({"empty": {}}) == {"empty": ""}

How it is used: This concept can be useful if you need to parse config files and simplify structures for father works. You can easily modify this for your own specification. And it's useful skill to read code and search bugs.

Precondition:
Keys in a dictionary are strings.
Values in a dictionary are strings or dicts.
root_dictionary != {}
